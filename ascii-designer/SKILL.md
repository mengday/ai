# ASCII Designer

专业的ASCII界面原型设计工具，能够根据需求描述或字段列表，快速生成软件界面、安装向导、网页布局及移动端App线框图。

**核心特性**：
- 自动保存设计稿到项目根目录的 `ascii-ui-design/` 文件夹
- 支持多种界面风格和布局模式
- 提供完整的设计说明和文档

---

## 工作流程 (Workflow)

本技能遵循以下工作流程：

```
用户需求 → 生成 ASCII 设计稿 → 自动保存到 ascii-ui-design/ → 输出设计说明
```

### 1. 需求分析
根据用户描述或字段列表，理解界面类型、功能需求和风格偏好

### 2. 设计生成
使用 ASCII 字符绘制界面线框图，遵循设计原则和组件规范

### 3. 自动保存
**重要**：每次生成设计稿后，必须自动保存到项目根目录的 `ascii-ui-design/` 文件夹
- 文件命名规则：`{页面名称}_{时间戳}.txt` 或 `{页面名称}_v{版本号}.txt`
- 示例：`login_page_20250112.txt`、`ticket_detail_v1.txt`
- 如果文件夹不存在，先创建文件夹

### 4. 设计说明
提供设计思路、关键元素、风格选择等说明文字

## 什么时候使用此技能 (Trigger Patterns)

**当出现以下情况时使用：**
- 用户需要快速绘制软件界面原型
- 用户提供了一组字段，要求生成界面布局
- 用户提到 "设计一个...界面"、"画一个...页面"
- 需要展示布局结构、信息层级，而不需要具体色彩
- 设计命令行界面 (CLI) 或终端 UI (TUI)
- 制作系统架构图的界面部分

**触发指令：**
- "设计一个登录页面"
- "生成一个工单详情页的 ASCII 图"
- "根据以下字段生成界面..."
- "帮我画一个 dashboard"

## 设计原则 (Design Principles)

1.  **清晰的视觉层级 (Visual Hierarchy)**
    -   使用 `╔═╗`、`┌─┐` 等不同粗细的边框区分层级
    -   重要信息（如标题、状态）置顶或加粗
    -   利用留白 (Whitespace) 分割内容块

2.  **一致的组件库 (Component Library)**
    -   **按钮**: `[ 确定 ]`、`[ 提交 ]`
    -   **输入框**: `┌──────────┐`
    -   **标签/Tag**: `[处理中]`、`[紧急]`
    -   **图标隐喻**: `[📄]` (文件), `[⚙️]` (设置), `[👤]` (用户), `[📎]` (附件)
    -   **复选/单选**: `[x]`, `(•)`

3.  **布局模式 (Layout Patterns)**
    -   **沉浸式头部**: 顶部大区域展示关键状态/背景
    -   **悬浮卡片**: 内容区域使用卡片容器 `┌─┐` 覆盖在背景上
    -   **固定底部**: 底部操作栏 `[ 按钮 ]` 固定在最下方
    -   **栅格系统**: 使用 `|` 分割左右或多列内容

## 界面类型与风格 (Styles)

| 风格名称 | 特点 | 适用场景 |
| :--- | :--- | :--- |
| **标准/沉浸式 (Immersive)** | 强调头部，信息层级分明，视觉舒适 | 移动端 App 详情页、个人中心 |
| **模块化/卡片式 (Modular)** | 信息分组清晰，块状分割 | 设置页、表单页、信息聚合页 |
| **紧凑/看板式 (Compact)** | 高密度信息，多列排版，减少滚动 | 运维后台、数据监控、CLI 工具 |
| **极简 (Minimalist)** | 最少线条，大量留白 | 现代 Web 首页、阅读类应用 |

## 输出规范 (Output Standards)

### 必须遵循的输出步骤

1.  **生成 ASCII 设计稿**
    - 始终使用 \`\`\`text 代码块包裹 ASCII 图，防止格式错乱
    - 确保字符对齐，使用等宽字符
    - 适当使用空行和分隔符增强可读性

2.  **保存到文件** ⭐ **必须执行**
    - 自动将设计稿保存到 `{项目根目录}/ascii-ui-design/` 文件夹
    - 文件命名格式：`{页面名称}_{YYYYMMDD}.txt` 或 `{页面名称}_v{N}.txt`
    - 文件内容包括：
        - 设计稿 ASCII 图
        - 设计说明
        - 创建日期和版本信息
    - **示例文件结构**：
      ```text
      # iOS 登录界面设计稿
      # 创建时间: 2025-01-12
      # 版本: v1.0

      [ASCII 设计稿内容]

      ---
      设计说明:
      - 布局风格: iOS 沉浸式布局
      - 登录方式: 账号密码、微信、Face ID、手势
      - 关键特性: ...
      ```

3.  **提供设计说明**
    - 在终端输出设计说明，解释布局思路、关键元素和风格选择
    - 说明设计稿保存路径
    - **示例**：
      ```text
      ✅ 设计稿已保存到: /path/to/project/ascii-ui-design/ios_login_20250112.txt

      📋 设计说明:
      - 布局风格: 采用 iOS 标准的沉浸式布局
      - 登录方式层级: 账号密码为主，快捷登录为辅
      - 关键特性: 支持生物识别、手势密码
      ```

4.  **可访问性与兼容性**
    - 尽量使用通用字符，避免使用仅特定字体支持的生僻符号
    - 提供响应式提示：如果设计包含复杂的宽表格，提示用户"建议在宽屏或全屏模式下查看"

5.  **设计稿文件模板**
    ```text
      # ============================================================
      # {页面名称} ASCII 设计稿
      # ============================================================
      # 创建时间: {YYYY-MM-DD HH:mm}
      # 版本: {v1.0}
      # 设计师: AI Assistant
      # ============================================================

      {ASCII 设计稿内容}

      # ------------------------------------------------------------
      # 设计说明
      # ------------------------------------------------------------
      ## 布局思路
      - {说明}

      ## 关键元素
      - {元素1}: {说明}
      - {元素2}: {说明}

      ## 风格选择
      - {风格}: {原因}

      ## 平台特性
      - iOS: {特性说明}
  ```

## 文件组织 (File Organization)

### 推荐的项目结构
```
项目根目录/
├── ascii-ui-design/           # 所有设计稿存放目录
│   ├── login/                 # 登录相关设计
│   │   ├── ios_login_v1.txt

│   ├── dashboard/             # 仪表板设计
```

### 版本管理建议
- 使用 `_v1`, `_v2` 后缀管理迭代版本
- 重大变更使用新文件名，小幅修改使用版本后缀
- 定期归档旧版本到 `archive/` 子目录
